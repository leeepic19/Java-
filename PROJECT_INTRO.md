# 网上订餐系统 (WSDC) 项目实践指导手册

## 1. 项目概述
本项目是一个基于 Java Web 技术的**网上订餐系统**。采用经典的 JSP + Servlet + JavaBean + JDBC 技术栈，实现一个完整的在线点餐功能。项目旨在模拟真实的订餐流程，包括用户注册登录、菜品浏览、购物车管理、订单支付以及后台管理等核心模块。

## 2. 项目目标
*   构建一个功能完善、界面友好的网上订餐平台。
*   掌握 Java Web 开发核心技术（Servlet, JSP, JDBC, Maven）。
*   熟悉软件开发流程，包括需求分析、数据库设计、编码实现、版本控制（Git）等。
*   培养团队协作能力，规范代码风格与项目结构。

## 3. 核心功能需求

### 3.1 前台用户端
1.  **用户注册与登录**
    *   用户必须登录账号才能进入选餐主界面。
    *   支持新用户注册。
    *   *技术要求*：登录参数加密（MD5+盐），支持验证码，使用 AJAX 异步提交。
2.  **主界面（选餐大厅）**
    *   展示所有菜品列表。
    *   支持菜品分类浏览。
    *   *技术要求*：实现分页显示功能。
3.  **菜品详情**
    *   点击主界面商品，弹出或跳转至详情页。
    *   展示菜品大图、价格、详细介绍等信息。
4.  **购物车管理（重点模块）**
    *   在主界面或详情页将商品加入购物车。
    *   购物车界面展示已选商品、数量、单价及总价。
    *   支持增加、减少数量或删除商品。
    *   *设计要求*：使用 `HashMap<String, CartItemBean>` 存储购物车数据，并存放于 `HttpSession` 中。
5.  **订单结算与支付**
    *   点击购物车结算，生成订单。
    *   **支付界面**：模拟支付流程，弹出二维码（集成微信/支付宝沙箱环境）。
6.  **个人中心**
    *   查看历史订单状态。
    *   修改个人信息。

### 3.2 后台管理端
1.  **管理员登录**：安全验证。
2.  **数据管理**：
    *   **菜品管理**：增删改查（上传图片、设置价格）。
    *   **订单管理**：查看订单状态，处理发货/完成。
    *   **用户管理**：查看和管理注册用户。
    *   **公告管理**：发布系统公告。

## 4. 技术架构与要求

### 4.1 技术栈
*   **前端**：HTML, CSS, JavaScript, jQuery, JSP, Bootstrap (推荐)
*   **后端**：Java Servlet
*   **数据库**：SQL Server 2022 (或 MySQL)
*   **连接池**：DBCP / C3P0 / Druid
*   **构建工具**：Maven
*   **版本控制**：Git

### 4.2 开发规范
*   **项目结构**：Maven 标准目录结构。
*   **分层架构**：
    *   `web` (Servlet/Controller)
    *   `service` (业务逻辑)
    *   `dao` (数据访问)
    *   `bean/vo` (实体类)
*   **DAO 层要求**：使用 Spring `JdbcTemplate` 重写 DAO 层（替代原始 JDBC）。
*   **代码规范**：统一命名、注释和缩进风格。

## 5. 数据库设计
*   **UserInfo** (用户表): ID, 用户名, 密码, 真实姓名, 邮箱等。
*   **Menu** (菜品表): ID, 菜名, 简介, 价格, 图片路径等。
*   **Orders** (订单表): ID, 用户ID, 菜品ID, 数量, 状态等。
*   **Admin** (管理员表): ID, 用户名, 密码。
